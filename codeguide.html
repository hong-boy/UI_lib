JavaScript

语言规范

变量（var）

每个变量声明都必须加上关键字var

否则该变量就会暴露在全局作用域（window）中，很可能覆盖全局作用域中的同名变量，从而引发问题。

常量

常量命名用类似NAMES_LIKE_THIS的形式。可以用@const来标记它是常量，但永远不要用const关键字来进行常量声明

分号

每一语句的结尾都要加上分号！ 否则会在调试、压缩、合并的时候引起不便甚至错误

块内函数声明

禁止使用块内函数声明

if (x) { function foo() {} }

如果一定要用的话可以在块内用变量来定义函数：

if (x) { var foo = function() {} }

异常处理

预先处理一些可能出现的异常也是有必要的（使用try-catch(e)）

如果是已知代码会发生错误时，不合适继续此条，而是应该预先对其进行检查，防止错误的出现。

自定义异常

支持合理使用，以提高代码可维护性.

没有自定义异常抛出的情况下，代码运行可能会报一些原始的错误信息，但包含的内容和错误描述会因浏览器而不同却都不是特别明确，不易维护

标准特性

总是优越于非标准特性之上。为了获得最大的可移植性和兼容性，尽量依赖于标准方法。

比如使用 string.charAt(3) 而非 string[3] ；比如通过DOM原生方法去访问节点元素，而非框架封装方法。

基本包装类型

禁止 new 基本包装类型！

typeof Boolean(0) == 'boolean';
typeof new Boolean(0) == 'object';

方法定义

推荐Foo.prototype.bar = function() { ... };

避免使用 Eval, With

避免使用Eval, With

闭包

推荐使用，但须注意避免在给DOM元素附加闭包时，可能产生的循环引用，从而造成内存消耗。例如

function foo(element, a, b) { element.onclick = function() { /* uses a and b */ }; }

即便这个闭包函数内部并没有使用 element，可它还是保持了对 element, a 和 b 的引用，而 element 也保持了对这闭包函数的引用，就导致了循环引用，无法被GC回收。如果遇到了这种情况，可以将代码优化如下

function foo(element, a, b) { element.onclick = bar(a, b); }
function bar(a, b) { return function() { /* uses a and b */ } }

this

仅在构造函数，方法，闭包中去使用它。

for-in 循环

只在 object/map/hash 要遍历键值的时候使用。

关联数组

永远不要用 Array 去做 map/hash/associative 要做的事情。

多行字符串字面量

禁止使用

空白字符开头字符行不能被很安全的编译剥离，以至斜杠后面的空格可能会产生奇怪的错误。虽然大多数脚本引擎都支持，但它并不是ECMAScript标准的一部分。所以如下代码可改为+连接字符串。

var str = 'A rather long string of English text, an error message \
     actually that just keeps going and going -- an error.'
var str = 'A rather long string of English text, an error message ' +
     'actually that just keeps going and going -- an error.'

Array和Object字面量

推荐使用

推荐使用 var arr = [1,2,3]; var obj = {name:'sth'};

而非 var arr = new Array(x1, x2); var o2 = new Object();

修改内置对象的原型

禁止

禁止修改 Object.prototype 和 Array.prototype 这样的内置对象。

风格规范

命名

通常，使用驼峰式命名。例如functionNamesLikeThis, variableNamesLikeThis, ClassNamesLikeThis, EnumNamesLikeThis, methodNamesLikeThis, 和 SYMBOLIC_CONSTANTS_LIKE_THIS

延迟初始

没必要总是在声明变量的时候就进行初始化，延迟初始化挺好的。

明确作用域

始终需要明确。 任何时候都要明确作用域 - 提高可移植性和清晰度。

括号

只在需要的时候使用。

对于一元操作符（如 delete, typeof 和 void）或是某些关键字（如return, throw, case, new）后面永远不要使用括号。

字符串

推荐单引号（'），特别是当创建一个HTML代码的字符串时候

注释

最好的注释是自解释的命名，即一个能清晰说明自身用途的命名；如果必须额外注释，必须写明具体意图。

拒绝一切为了注释而注释（比如对代码命名的简单重复）

内部类和枚举

内部类和枚举的定义应和顶级类放在同一文件内。只需要声明顶级类。

编译压缩

推荐使用。 使用被编译压缩后的JS代码可以更快下载和使用
